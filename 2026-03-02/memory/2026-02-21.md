### 完成的功能
1. **红包消息过滤** — v2.0.7: 插件端过滤 wxpay://、type=2001/2000，不再触发AI
2. **群成员数同步** — v2.0.7: 通过 CContact.m_nsChatRoomMemList 计数，上报到服务端，网页显示
3. **Per-group @触发开关** — groups 表加 require_at 字段，每个群独立控制@触发/全部消息
4. **AI 触发移到服务端** — v2.1.0: 插件不再调 /api/ai/chat，服务端 /api/messages 收到消息后自动判断是否触发AI，减少一半网络请求
5. **防自我循环** — 过滤 bot 自己的消息：wxid_wp5s0cadkm4h22 + "self" + 空字符串 + 昵称"多多"
6. **AI 群信息上下文** — callClaude 前注入【当前环境】：群名、群ID、群人数、发送者信息
7. **@多多精确匹配** — 服务端检查 message.includes('@多多') 而不是任意 @
8. **图片路径修复** — path.resolve 绝对路径替代 path.join 相对路径
9. **消息合并缓冲区** — 4秒窗口，同一人连续消息/图片+文字合并后再发AI，更自然
10. **图片重复触发修复** — v2.1.1: 删除插件端图片AI触发，全交给服务端 update-media

### 踩的坑（经验沉淀）
- **sql.js ALTER TABLE 不影响运行中进程** — sql.js 启动时加载DB到内存，外部改了文件进程不知道。必须重启PM2 + 在 initDb 里用 safeAddColumn 自动迁移
- **sed 不能改中文** — sed 处理 UTF-8 中文会变乱码，用 node 脚本改文件才安全
- **path.join 与绝对路径** — `path.join(dir, '..', '..', '/uploads/x')` 不会得到期望的绝对路径，用 `path.resolve(baseDir, '.' + urlPath)` 代替
- **enrichedMessage 破坏 URL 解析** — callClaude 收到的 message 后面附加了上下文，imageUrl = message.replace('[图片] ','').trim() 会把上下文也带上，必须 .split('\n')[0] 只取第一行
- **bot 自己发的消息 real_sender 是空字符串** — 不是 wxid 也不是 "self"，是 ""。过滤条件必须包含空字符串
- **msg_type 有两种格式** — 插件上报时可能是数字 "3" 或文字 "image"，skipTypes 两种都要包含
- **require_at 检查被插件端剥离** — 插件把 @多多 从消息里去掉再发服务端，服务端 message.includes('@多多') 永远为 false。解决：插件额外发 raw_message 保留原始内容
- **processAIMessage 后必须 return** — 否则旧代码继续执行报 ReferenceError

### 架构变化
- **之前**: 插件发2次请求（/api/messages + /api/ai/chat）
- **现在**: 插件只发1次（/api/messages），服务端自动判断触发AI
- **消息缓冲区**: messages.js 里 msgBuffer Map，4秒窗口合并同一人连续消息
- **图片AI**: update-media 路由里触发，2秒延迟等文件写完，走缓冲区合并
