## 技术细节

### 插件端 (WeChatMonitor.dylib v1.2.0)
- 编译方式: macOS clang → vtool 改 platform 为 iOS → install_name_tool 修 framework 路径
- 无 iphoneos SDK，用 `-undefined dynamic_lookup` + macOS SDK 交叉编译
- dylib 放在 `Frameworks/` 目录，注入路径 `@executable_path/Frameworks/WeChatMonitor.dylib`
- 基于参考 IPA (多开版，含 libsubstrate/libellekit 基础设施)
- Hook 3 个方法: AsyncOnAddMsg + AsyncOnPreAddMsg + AddMsg
- 用 dlsym 动态查找 MSHookMessageEx，fallback 到 method_setImplementation
- 通过 WCPluginsMgr 的 `registerControllerWithTitle:version:controller:` 注册到插件列表
  - controller 参数是**类名字符串**，不是实例！(之前传实例导致 isEqualToString 崩溃)
- UI: 纯 objc_msgSend 动态创建 UITableViewController 子类
- 网络: NSURLSession POST 上报消息到服务器

### 关键闪退教训
1. dylib 注入路径必须是相对路径 `@executable_path/...`，不能是绝对路径
2. macOS 编译的 dylib 需要修 Foundation/CoreFoundation 路径去掉 `Versions/C/`
3. WCPluginsMgr registerController 的 controller 参数是 NSString 类名

### 经验教训
- **编译检查**: `grep "error:"` 返回 exit code 1 (无匹配) 不代表编译成功，要看 dylib 文件是否生成
- **sql.js 没有 db.exec()**: 用 `getDb().run()` 执行 DDL，或 `db.prepare().run()` 但注意空参数
- **延迟建表**: sql.js 的 db 是 async 初始化的，路由文件加载时可能还没准备好，用 lazy init
- **OpenClaw gateway 没有外部 sessions REST API，但有 webhook 端点**: sessions_send 只能内部调用，但 `/hooks/agent` 和 `/hooks/wake` 可供外部程序 POST 触发 agent 任务（需配置 `hooks.enabled + hooks.token`）。文档: https://docs.openclaw.ai/automation/webhook
