## 微信插件图片处理问题排查 (v1.8.2 → v1.8.5)

### 问题
图片消息在网页端始终显示"image_loading"，图片无法加载

### 根因
`AsyncOnAddMsg:MsgWrap:` 时 wrap 对象的 `m_nsFilePath` 和 `m_nsThumbImgPath` 都是 nil——图片还没下载，路径还没被微信填充。即使 retain wrap 对象，延迟后读路径仍然是 nil（wrap 的属性不会被更新）

### 解决方案 (v1.8.5)
照抄老林 WGMImageHandler 的方式：
1. 同步提取 `m_uiMesLocalID` + `chatName`（不保留 wrap 引用）
2. 触发下载：`StartDownloadImage:HD:NO/YES AutoDownload:YES`
3. 多轮重试时通过 `CMessageMgr.GetMsg:LocalID:` 从数据库重新获取消息对象
4. 从新获取的消息读路径 → WXAM 解密 → 上传

### 经验沉淀
- **遇到X就做Y**: 微信 wrap 对象的路径属性在 AsyncOnAddMsg 阶段是 nil → 必须用 GetMsg:LocalID: 从数据库重新获取
- **retain wrap 没用**: wrap 的属性不会被异步更新，必须重新从 CMessageMgr 获取 fresh 对象
- v1.8.5 第一次重试会列出 CMessageMgr 的所有 GetMsg 相关方法，等待测试结果

### Token 用量
- 云寒问了 token 用量，当天 session 压缩了 7 次，粗估 100-150 万 input tokens

## 图片功能完整复盘 (v1.8.1 → v1.9.7，约 18 个版本)

### 最终成功方案 (v1.9.7)
1. **触发下载**: `StartDownloadImage:HD:NO/YES AutoDownload:YES`
2. **获取路径**: `CMessageMgr.GetMsg:LocalID:` 从数据库重新获取消息对象，读 `m_nsThumbImgPath`
3. **找文件**: 优先缩略图 `.pic_thum`(JPEG)，fallback `.pic`(WXAM)
4. **WXAM 解码**: 主线程 `WxAMAnimatedImageDecoder.imageWithWxAMData:scale:config:` → `UIImageJPEGRepresentation`
5. **上传**: base64 JSON POST `/api/upload`
6. **更新**: `postJSON("/api/messages/update-media", {timestamp, media_url})`

### 踩坑清单（遇到 X 就做 Y）

#### ObjC 类型转换（最致命的坑）
- ❌ `(long)msg[@"timestamp"]` — 这是把 NSNumber **指针地址**转 long，不是取值！
- ✅ `[msg[@"timestamp"] longValue]` — 正确取 NSNumber 的数值
- **规则**: ObjC 容器（NSDictionary/NSArray）里存的都是对象，取数值永远用 `xxxValue` 方法

#### AsyncOnAddMsg 时 wrap 的属性
- ❌ `wrap.m_nsFilePath` / `wrap.m_nsThumbImgPath` — 在 AsyncOnAddMsg 阶段为 nil
- ❌ retain wrap 后延迟读 — 属性仍然不会被更新
- ✅ 同步提取 `m_uiMesLocalID` + `chatName`，延迟后用 `GetMsg:LocalID:` 从数据库重新获取

#### WXAM 解码
- ❌ 在后台线程调用 `WxAMAnimatedImageDecoder` — EXC_BAD_ACCESS 闪退
- ❌ NSInvocation + `__unsafe_unretained` + `getReturnValue` — ARC 下野指针
- ❌ `dlclose(UIKit handle)` — UIKit 是常驻框架，dlclose 可能影响返回值内存
- ✅ **主线程** `dispatch_async(dispatch_get_main_queue())` + semaphore 等待
- ✅ `objc_msgSend` 直接调用（不用 NSInvocation）
- ✅ `[jpegData copy]` 确保数据不被 ARC 提前释放
- ✅ 不 dlclose UIKit
- 方法签名: `imageWithWxAMData:scale:config:` (3 参数) 和 `imageWithWxAMData:scale:` (2 参数) 都存在，优先用 3 参数版本

#### 上传方式
- ❌ multipart/form-data 在 iOS 上 NSURLSession 发送大 body 会闪退
- ✅ base64 JSON POST 稳定可靠
- 服务端同时支持 multipart (multer) 和 base64 JSON

#### 微信图片文件格式
- `.pic_thum` = 标准 JPEG（缩略图，可直接上传显示）
- `.pic` = WXAM 加密格式（magic: 0x77786766 "wxgf"，需要解码）
- `.pic_hd` = WXAM 加密格式（高清版）
- WXAM 不是 JPEG/PNG/WebP 的变种，是微信自定义格式，无法被浏览器直接显示
