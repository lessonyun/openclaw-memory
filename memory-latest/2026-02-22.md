## 🎤 语音发送功能开发 (16:00-17:20)

### 最终成功方案 (v2.1.7)
1. edge-tts 生成 MP3（服务器端）
2. ffmpeg MP3→PCM (24000Hz, mono, s16le)
3. silk-wasm PCM→SILK（已含 0x02 前缀，不要再加）
4. base64 编码 → 命令队列 → 插件取执行
5. 插件端：
   - CMessageWrap initWithMsgType:34
   - setM_nsFromUsr: 自己wxid
   - setM_nsToUsr: target
   - setM_dtVoice: silkData
   - setM_uiVoiceTime: duration(ms)
   - **setM_uiVoiceFormat: 4**（不是0！抓包得知）
   - setM_uiVoiceEndFlag: 1
   - setM_uiStatus: 1
   - **setM_nsContent: XML格式** `<msg><voicemsg voicelength="xxx" voiceformat="4" forwardflag="0" /></msg>`
   - AddMsg:MsgWrap: 插入消息
   - 文件名用 **localId.aud** 保存到 Documents/Audio/
   - **SaveMesVoice:MsgWrap:** 保存语音到数据库
   - **AudioSender ResendVoiceMsg:MsgWrap:** 触发上传
   - **AudioSender SendOriVoiceMsgWithUserData:** 也调用

### 关键踩坑
- **silk-wasm 输出已含 0x02 前缀** → 不要再 Buffer.concat([0x02], silk.data)，否则双前缀
- **服务器(国内)连不上 Fish Audio** → 改用 edge-tts（微软免费TTS）
- **CMessageMgr 没有 ResendVoiceMsg** → 在 AudioSender 上（通过 getService 获取）
- **MMNewUploadVoiceMgr getService 返回 nil** → 不可用
- **voiceFormat=0 不对** → 正常录音用 voiceFormat=4
- **content 不能是文件路径** → 必须是 XML 格式
- **文件名要用 localId.aud** → 先 AddMsg 拿到 localId 再保存文件

### 核心经验（再次验证）
- **抓包对比 > 盲猜**：hook AddMsg 对比手动发语音和代码发语音的 wrap 差异，一次对比解决所有问题
- **20次盲猜不如1次对比**：从 v2.1.1 到 v2.1.6 全是盲猜，v2.1.7 一次对比就成功
- 正常语音发送时 AddMsg 阶段 m_dtVoice 是空的（streaming 上传），但我们设置了也能工作

- edge-tts → ffmpeg → silk-wasm → 命令队列
