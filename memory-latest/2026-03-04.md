**设备配对 (pairing)**
- OpenClaw 要求浏览器设备先配对才能连接
- 批准命令：`docker exec {agent} openclaw devices approve {requestId}`
- 列出 pending：`docker exec {agent} openclaw devices list`
- 批准脚本：`lobster-squad/approve.sh`

**踩坑经验（重要！）**
1. HTTP + 非 localhost = 浏览器砍掉 Web Crypto API → pairing 无法工作
2. 解决方案只有三条路：Chrome flag / HTTPS / localhost
3. Chrome flag 是最简方案：一次设置，永久生效，零开销
4. `auth=none` + `bind=lan` 被 OpenClaw 拒绝
5. `auth=trusted-proxy` 需要额外 trustedProxy 配置
6. `requirePairing`/`skipPairing` 不是合法配置项
7. 多个 localhost 端口共享同一个 localStorage（同源策略）→ token 互相覆盖
8. `.localhost` 子域名可解决同源问题（Chrome 自动解析到 127.0.0.1 + 安全上下文）
9. 认证失败太多次会被限速（"too many failed attempts"），重启容器可清除
